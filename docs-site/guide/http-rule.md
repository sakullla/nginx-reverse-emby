# HTTP 反向代理

这篇假设你已经完成 [部署](./deploy.md)，并能打开面板。HTTP 规则按域名反代 Web 服务，理解前端 URL、后端 URL、重定向和代理头，适合浏览器、电视盒子和 Emby / Jellyfin 客户端。

一个典型规则形如：

```text
客户端访问: https://app.example.com
节点代理到: http://192.168.1.100:8096
```

把 `app.example.com` 换成解析到节点的域名，把后端换成你的实际服务地址（含协议、域名、端口）。

## 1. 登录面板

打开：

```text
http://<服务器 IP>:8080
```

输入 `docker-compose.yaml` 里设置的 `API_TOKEN`。

![登录面板](/screenshots/panel-login.png)

## 2. 进入 HTTP 规则页面

登录后，在左侧导航 **流量管理 → HTTP 规则**。页面顶部如有节点选择，先选择 `local` 或你要承载规则的节点。

![HTTP 规则页面](/screenshots/panel-http-rules.png)

## 3. 添加规则

点击 **添加规则**，表单分为 **基础配置**、**高级配置**、**Relay 配置** 三个标签页。

### 基础配置

| 字段 | 示例 | 怎么填 |
| --- | --- | --- |
| 前端访问地址 | `app.example.com` | 选择 `http://` 或 `https://`，右侧填入口域名。 |
| 后端地址 | `http://192.168.1.100:8096` | 选源站实际协议，右侧填后端域名 / IP 和端口。 |

![添加 HTTP 规则](/screenshots/panel-http-rule-form.png)

下方可选填标签、出口 Profile，并保持 **启用规则** 开启。

### 高级配置

控制出口路径、负载均衡、重定向和客户端 IP 透传行为：

| 选项 | 说明 |
| --- | --- |
| 代理 302/307 重定向 | 开启时，后端返回的 `Location` 会被重写为前端地址；关闭时原样透传给客户端。默认开启。 |
| 出口 Profile | 选择出站经过的 Egress Profile（如 SOCKS / HTTP 代理、WireGuard 出口）。`Direct` 表示直连。 |
| 请求头 | 可按需自定义发给后端的请求头。 |

### Relay 配置

如需让请求经中继隧道再到后端，在 **Relay 配置** 标签页添加 Relay 层。Relay 的概念见 [L4 规则与 Relay](./l4-relay.md) 和 [Relay 参考](../reference/relay.md)。

## 4. 保存并启用

保持 **启用规则** 开启，点击 **创建规则**。回到列表能看到新规则即表示面板已记录成功；`local` agent 会按心跳同步配置并开始监听。

## 5. 配好 DNS 和端口

规则保存后还需确认外部访问条件：

- 入口域名已解析到节点公网 IP。
- 安全组和防火墙已放行访问端口（`http://` 通常 `80`，`https://` 通常 `443`）。
- 节点能访问后端服务。
- 若后端需要登录、邀请码或账号，这些仍按后端规则处理，反代不会绕过权限。

## 6. 启用 HTTPS（可选）

新手优先使用 HTTP-01 自动签发证书，无需配置 Cloudflare Token。把前端访问地址改成 `https://app.example.com`，然后确认：

- 域名已解析到节点。
- 节点 `80`、`443` 端口已放行。
- 规则保存后等待 Agent 同步并自动申请证书。

HTTP-01 会临时占用 `80` 端口完成域名验证。若节点不能开放 `80` 端口或需要通配符证书，改用 DNS 验证，详见 [证书与 HTTPS](./certificates.md)。

## 7. 浏览器验证

在浏览器打开入口地址：

```text
http://app.example.com
```

能看到目标页面即说明规则生效。

## 流式恢复与重试

HTTP 代理内置中断流恢复和同 backend 安全重试，用来改善大文件播放 / 下载时的偶发断流。它不会绕过后端权限，也不会把不可达的后端变成可用后端。

中断恢复覆盖：

- `GET` 全量响应。
- `GET` 单区间 `Range` 响应。

后端需要持续支持 `Accept-Ranges: bytes`，并返回稳定的强 `ETag` 或 `Last-Modified`。当校验器或 `Content-Range` 不一致时，恢复会停止并返回错误。相关变量见 [环境变量](../reference/environment.md)。

## 何时关闭 302/307 改写

以下场景可在规则的 **代理 302/307 重定向** 处关闭：

- CDN 回源需要保留原始重定向地址。
- 多跳转链接需要客户端直接访问。
- OAuth 回调需要重定向地址原样传递。

## 排查清单

打不开时按此顺序排查：

1. 域名是否解析到正确节点。
2. 节点端口是否放行。
3. 节点是否能访问后端（`curl -I <后端地址>`）。
4. 规则是否选择了正确 Agent。
5. 规则是否保持启用。
6. 后端是否限制 Host、地区、IP 或需要特殊路径。

| 现象 | 常见原因 |
| --- | --- |
| 打开域名没反应 | DNS 没生效，或端口没放行。 |
| 规则保存了但访问不到后端 | 节点无法访问后端，或后端协议、端口填错。 |
| HTTPS 访问失败 | 还未配置证书，先用 HTTP 验证链路正常后再启用 HTTPS。 |
| 登录页能打开但播放失败 | 节点到后端线路不稳、后端限制反代，或客户端仍在请求后端直链。 |
| 代理到错误服务 | 前端或后端地址填错，检查协议、域名、端口。 |
