# 中继参考

中继隧道将流量从一个代理传输到另一个代理。它们通常与 HTTP 规则或 L4 规则结合使用：规则定义公共入口点和最终后端，而中继监听器定义代理之间的中间路径。

## 支持的传输模式

| 模式 | 说明 |
|------|------|
| `tls_tcp` | TLS over TCP。默认且最容易配置的模式。 |
| `quic` | QUIC over UDP。更低的握手延迟；如果 QUIC 不可用，可以回退到 TLS/TCP。 |
| `wireguard` | 中继隧道通过受管理的 WireGuard 接口传输。 |

## 监听器字段

在面板中创建中继监听器时，你需要填写以下字段：

| 字段 | 含义 |
|------|------|
| **名称** | 在面板中的显示名称。 |
| **证书来源** | 默认是自动签发（中继 CA）。你也可以绑定现有证书。 |
| **绑定地址** | 代理上监听的本地地址。每行一个。 |
| **监听端口** | 代理上监听的本地端口。 |
| **公共入口** | 其他节点用来连接此中继的地址。格式：`host` 或 `host:port`。 |
| **中继传输** | 选择 `TLS/TCP`、`QUIC` 或 `WireGuard`。 |
| **信任策略** | 默认是自动中继 CA + 固定。你可以在高级设置中自定义。 |
| **标签** | 用于筛选节点的标签。 |

## 传输模式详情

| 模式 | 何时使用 |
|------|----------|
| `tls_tcp` | 从这里开始。它在大多数网络中都能工作，且配置最简单。 |
| `quic` | 当你想要更低延迟，且防火墙允许 UDP/QUIC 时使用。 |
| `wireguard` | 当中继路径必须通过受管理的 WireGuard VPN 时使用。 |

`wireguard` 模式只改变传输路径。中继仍然在其之上使用 TLS、多路复用和认证。

## TLS 和信任策略

默认情况下，中继监听器使用自动签发的中继 CA 证书加上证书固定。这是安全的，不需要手动配置。

如果你需要完全手动控制，打开高级设置并选择一种 TLS 模式：

| TLS 模式 | 含义 |
|----------|------|
| 固定 + CA | 同时验证证书固定和 CA 信任链（默认）。 |
| 仅固定 | 只验证证书固定。 |
| 仅 CA | 只验证 CA 信任链。 |
| 固定或 CA | 任一检查通过即接受。 |

你也可以提供**固定集**（每行一个，格式 `type:value`）和**受信任的 CA 证书**。

对于 `TLS/TCP` 模式，你可以选择性地选择 **TLS 混淆策略**（例如 `early_window_v2`）。这只适用于第一跳。

## 中继层

HTTP 规则和 L4 规则使用 `relay_layers` 来描述隧道路径。每一层可以包含多个中继监听器，它们作为并行候选。层按顺序处理：

```text
客户端 -> 第 1 层 -> 第 2 层 -> 后端
```

旧的 `relay_chain` 字段仅保留用于向后兼容。新配置应该使用 `relay_layers`。

## 超时变量

| 变量 | 默认值 | 作用 |
|------|--------|------|
| `NRE_RELAY_DIAL_TIMEOUT` | `5s` | 连接到上游中继节点的超时时间。 |
| `NRE_RELAY_HANDSHAKE_TIMEOUT` | `5s` | 中继握手的超时时间。 |
| `NRE_RELAY_FRAME_TIMEOUT` | `5s` | 读取或写入单个中继帧的超时时间。 |
| `NRE_RELAY_IDLE_TIMEOUT` | `2m` | 空闲中继连接保持打开的时间。 |
