# 流量统计与额度

控制面按 Agent 所在主机 / 网络命名空间的网卡累计计数，统计周期总量并据此做额度阻断。HTTP 规则、L4 规则和 Relay 监听器的代理侧统计保留为分项分析，不直接作为计费口径。

## 采集模型

- Linux Agent 优先通过 netlink 读取内核网卡计数，失败时回退到 `/proc/net/dev`。
- 读取 `/proc/sys/kernel/random/boot_id` 检测系统重启；网卡重建或计数回退时记录 counter reset，并把当前值作为新基线，避免产生负数或误扣历史周期。
- 控制面持久化 raw cursor、小时桶、日汇总、月汇总、节点策略、校准基线和事件。

## 网卡筛选

默认采集全部网卡。如果 Agent 运行在 Docker bridge 网络中，采到的是容器命名空间内的流量；需要 VPS 主机网卡总量时，建议使用 host 网络或主机部署，并用 `NRE_TRAFFIC_INTERFACES` 限定计入的网卡：

```ini
NRE_TRAFFIC_INTERFACES=eth0,eth1
```

## 计费周期与额度

每个 Agent 有一份流量策略，决定流量如何计入额度和何时阻断。主要字段：

| 字段 | 说明 |
| --- | --- |
| 统计方向 | 入站（rx）、出站（tx）、双向（both）、取最大值（max）、混合（mixed）。 |
| 计费起始日 | 每月计费周期的起始日（1–31），默认 `1`。 |
| 月度额度 | 月度流量上限，留空表示不限。 |
| 超额阻断 | 开启后，周期内累计达到额度会阻断该节点流量，并记录阻断原因。 |

阻断状态会体现在节点列表（`traffic_blocked`）和仪表盘的 **阻断 X / Y** 统计中。

## 数据保留

| 字段 | 默认值 | 说明 |
| --- | --- | --- |
| 小时桶保留 | 30 天 | 详细小时级数据保留天数。 |
| 日汇总保留 | 3 个月 | 日级汇总保留月数。 |
| 月汇总保留 | 24 个月 | 月级汇总保留月数。 |

控制面按 `NRE_TRAFFIC_CLEANUP_INTERVAL`（默认 `24h`）主动清理过期历史。

## 仪表盘

**首页** 仪表盘的流量模块展示：

- **阻断** 统计（已阻断 / 总节点数）。
- **已用 / 额度 / 剩余**（按所选节点聚合）。
- **流量分布**（额度环形图）。
- **计费周期**（方向、起始日、剩余预算）。
- **流量趋势**（可按小时 / 日 / 月切换，可选具体节点或全部）。
- **Top 规则** 与 **Top 节点** 排行。

## 手工校准

当物理计数与实际不符（例如更换网卡、重置系统、迁移数据），可在节点上执行校准：设置基线偏移，或清零当前周期数据。校准基线保存在 `traffic_baselines` 中，后续统计以校准后的值为口径。

## 关闭流量统计

如不需要流量统计或希望降低数据库写入量：

```ini
NRE_TRAFFIC_STATS_ENABLED=false
```

该变量同时作用于控制面和 Agent。相关变量完整清单见 [环境变量](./environment.md#流量统计)。
