# 流量统计与配额

控制面通过读取每个代理主机上的网络接口来跟踪网络使用情况。它统计进出字节数，执行月度配额，并可以在超出限制时阻止流量。

HTTP 规则、L4 规则和中继监听器也有自己的每条规则流量计数器用于分析，但这些不用于配额执行。

## 流量如何采集

| 步骤 | 发生了什么 |
|------|------------|
| 1. 读取计数器 | Linux 代理通过 **netlink** 读取内核网络接口计数器。如果失败，则回退到 `/proc/net/dev`。 |
| 2. 检测重启 | 代理读取 `/proc/sys/kernel/random/boot_id` 来检测系统重启。 |
| 3. 处理重置 | 如果接口被重新创建或其计数器回滚，代理会记录一次计数器重置，并使用当前值作为新的基准。这防止了负数或不正确的扣除。 |
| 4. 持久化数据 | 控制平面存储原始游标值、每小时桶、每日汇总、每月汇总、节点策略、校准基准和事件。 |

## 筛选网络接口

默认情况下，所有接口都会被监控。如果代理在 Docker 桥接网络中运行，它只能看到容器内部的流量。要测量主机 VPS 的总流量，请使用 **host 网络模式** 或直接在主机上部署代理，并限制监控到主机接口：

```ini
NRE_TRAFFIC_INTERFACES=eth0,eth1
```

## 计费周期与配额

每个代理都有一个流量策略，控制使用量如何统计以及何时阻止流量。

| 字段 | 含义 |
|------|------|
| **计数方向** | 哪些方向计入配额：入站（rx）、出站（tx）、双向、两者中的最大值，或混合公式。 |
| **计费起始日** | 月度周期重置的日期（1–31）。默认是 `1`。 |
| **月度配额** | 每月允许的最大流量。留空表示无限制。 |
| **超限时阻止** | 启用后，一旦当前周期达到配额，代理的流量就会被阻止。 |

当代理被阻止时，节点列表会显示 `traffic_blocked`，仪表盘显示一个 **已阻止 X / Y** 计数器。

## 数据保留

| 粒度 | 默认保留期 | 存储内容 |
|------|------------|----------|
| 每小时桶 | 30 天 | 详细的每小时流量数据。 |
| 每日汇总 | 3 个月 | 聚合的每日总量。 |
| 每月汇总 | 24 个月 | 聚合的每月总量。 |

控制平面根据 `NRE_TRAFFIC_CLEANUP_INTERVAL`（默认 `24h`）自动清理旧数据。

## 仪表盘

**首页**仪表盘显示一个流量模块，包含：

- **已阻止** 计数（已阻止节点 / 总节点）。
- **已用 / 配额 / 剩余**（跨选定节点聚合）。
- **配额分布**（环形图）。
- **计费周期**（方向、起始日、剩余预算）。
- **流量趋势**（可按小时 / 天 / 月切换，针对所有节点或特定节点）。
- **热门规则** 和 **热门节点** 排名。

## 手动校准

在以下事件后，物理计数器可能无法匹配实际使用量：

- 更换网卡
- 重置操作系统
- 将数据迁移到新服务器

你可以从节点详情页校准代理：

| 校准操作 | 效果 |
|----------|------|
| 设置基准偏移 | 从当前计数器增加或减去固定数量。 |
| 清除当前周期 | 将当前计费周期的使用量重置为零。 |

校准基准保存在 `traffic_baselines` 中。所有未来的统计都使用校准后的值。

## 禁用流量统计

如果你不需要流量跟踪或想要减少数据库写入：

```ini
NRE_TRAFFIC_STATS_ENABLED=false
```

此变量同时适用于控制平面和代理。完整的流量相关变量列表请参阅 [环境变量](./environment.md#流量统计)。
