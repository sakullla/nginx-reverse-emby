# 常见问题

这里整理了新手最常遇到的问题。如果找不到答案，请检查对应的功能文档页面。

## 为什么默认 Compose 文件使用 host 网络？

Docker 默认的 bridge 网络有一个限制：**容器运行后无法动态新增端口映射**。

而内嵌的 local agent 会根据你创建的规则动态创建监听端口。使用 host 网络后，这些端口可以直接绑定到宿主机上，无需预先在 Compose 中声明。

**简单说**：host 网络让端口管理更灵活，你不需要每次新增规则都改 Compose 文件。

## 面板数据存储在哪里？

默认 Docker Compose 部署中，数据存储在两个位置：

| 位置 | 路径 | 说明 |
| --- | --- | --- |
| 宿主机 | `./data` | Compose 文件同级目录下的 `data` 文件夹 |
| 容器内 | `/opt/nginx-reverse-emby/panel/data` | 容器内的挂载点 |

所有配置、证书、Agent 信息都保存在这里。**请定期备份这个目录。**

## 数据库可以切换吗？

可以。默认使用 SQLite（零配置、文件型），适合个人和小团队。

当数据量增大或需要多实例时，可以切换到 PostgreSQL 或 MySQL：

1. 设置环境变量：
   ```ini
   NRE_DATABASE_DRIVER=postgres
   NRE_DATABASE_DSN=postgres://nre:nre@localhost:5432/nre?sslmode=disable
   ```

2. 如果有现有数据，使用 `migrate-storage` 迁移：
   ```bash
   nre-control-plane migrate-storage --from-driver sqlite --from-dsn ... --to-driver postgres --to-dsn ...
   ```

详见 [迁移](./migration.md)。

## `local` 是什么？

`local` 是 Docker Compose 默认启用的**内嵌 local agent**。

它和控制面跑在同一个容器里，但监听端口绑定到宿主机网络。如果你只在一台 VPS 上做反代，所有规则都选 `local` 即可，不需要额外部署 Agent。

**什么时候需要远程 Agent？**
- 有多台服务器，需要分散代理负载
- 需要在不同地理位置提供入口
- 需要把内网服务暴露出来

## 远程 Agent 必须开放入站端口吗？

**不需要。**

Agent 使用**心跳拉取（heartbeat pull）**模式：Agent 主动连接控制面，获取配置并应用。只要 Agent 能访问控制面即可，即使 Agent 在 NAT 或内网后面也没问题。

**控制面不会主动连接 Agent。**

不过，代理规则本身的监听端口（如 80、443、或其他你配置的端口）仍需要按规则放行，否则外部访客无法访问。

## 为什么我访问还是要挂代理？

检查两点：

**1. 客户端配置是否正确**

确认你填的是**自己的入口域名**，而不是后端原始地址。例如：
- 正确：`https://myapp.yourdomain.com`（指向你的节点）
- 错误：`https://origin.example.net`（后端原始地址）

本项目的做法是：让你的节点去访问后端，客户端访问你的节点。

**2. 节点能否访问后端**

在节点上执行：

```bash
curl -I https://origin.example.net
```

如果节点到后端不通（返回错误、超时、或被防火墙阻断），规则保存成功也无法解决访问问题。你需要先确保网络连通。

## HTTP 规则和 L4 规则怎么选？

| 场景 | 选择 | 原因 |
| --- | --- | --- |
| 反代 Web 入口、Emby/Jellyfin、网站 | **HTTP 规则** | 支持按域名、路径、HTTP 头处理，支持重定向改写、代理头注入 |
| 转发 TCP/UDP 端口、游戏服务器、数据库 | **L4 规则** | 直接转发流量，不关心 HTTP 内容 |
| 接入 Relay 隧道 | **L4 规则** | Relay 链路在传输层工作 |

**HTTP 规则** 适合浏览器、电视盒子、Emby / Jellyfin 客户端 — 它能理解 URL、处理重定向、注入代理头。

**L4 规则** 适合任何 TCP/UDP 流量 — 它只是把流量从 A 点搬到 B 点，不关心里面是什么协议。

## 何时关闭 302/307 改写？

HTTP 规则默认会改写 302/307 重定向响应，把其中的地址换成你的入口域名。这在大多数情况下是正确的行为。

以下场景需要关闭：

- **CDN 回源** — 需要保留原始重定向地址，不能改写
- **多跳转链接** — 客户端需要直接访问中间地址
- **OAuth 回调** — 重定向地址需要原样传递给 OAuth 提供商

关闭位置：HTTP 规则的 **代理 302/307 重定向** 选项。

## 证书需要手动申请吗？

**不需要。**

HTTP 规则保存后，Agent 会在同步配置时自动用 HTTP-01 方式申请证书。整个过程无需人工干预。

**什么时候需要手动配置？**
- 无法开放 80 端口 → 配置 Cloudflare DNS-01
- 需要通配符证书（`*.example.com`）→ 配置 DNS-01
- 已有商业证书 → 手动上传

详见 [证书与 HTTPS](../guide/certificates.md)。

## 流量额度超了会怎样？

如果你在 **流量策略** 中开启了 **超额阻断**：

- 节点在计费周期内累计达到月度额度时会被**阻断**
- 被阻断的节点在节点列表和仪表盘中会显示阻断状态
- 阻断期间该节点不再接受新连接

阻断是**周期性的** — 下个计费周期开始后会自动恢复。你也可以手动调整额度或关闭超额阻断。

详见 [流量统计与额度](../reference/traffic.md)。

## 如何升级 Agent？

在 **版本策略** 页面创建策略：

1. 设置 `desired_version`（目标版本号）
2. 上传各平台的安装包（URL + SHA256 校验值）
3. 保存策略

Agent 在每次心跳时会收到策略信息，然后自动：
- 下载新版本安装包
- 校验 SHA256
- 原子更新（先下载完成再切换，失败自动回滚）

详见 [开发与构建](../reference/development.md#版本更新)。

## 根目录 docs 还能继续放临时文档吗？

可以。

公开文档站源码位于 `docs-site/` 目录，构建后会发布为文档网站。根目录 `docs/` 可以继续放：
- 临时设计文档
- 开发计划
- 内部技术文档
- 不面向用户的材料

两者互不冲突。
