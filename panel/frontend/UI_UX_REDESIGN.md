# UI/UX 重设计文档

## 概述

本次 UI/UX 重设计对 Nginx Reverse Proxy 管理面板进行了全面的现代化改造,提升了视觉效果、用户体验和交互质量。

## 设计理念

- **现代化** - 采用最新的设计趋势和视觉风格
- **简洁性** - 清晰的信息层次和简洁的界面
- **一致性** - 统一的设计语言和交互模式
- **响应式** - 完美适配各种屏幕尺寸
- **可访问性** - 良好的对比度和可读性

## 主要改进

### 1. 设计系统升级

#### 颜色系统
- **主色调**: 从紫色 (#8b5cf6) 更新为现代蓝紫色 (#6366f1)
- **新增颜色**: 添加了完整的语义化颜色系统
  - Success (成功): #10b981
  - Warning (警告): #f59e0b
  - Danger (危险): #ef4444
  - Info (信息): #3b82f6
  - Secondary (次要): #14b8a6

#### 间距系统
- 采用 8px 基准的间距系统
- 从 xs (8px) 到 5xl (80px) 的完整间距等级

#### 阴影系统
- 多层次阴影系统 (xs, sm, md, lg, xl, 2xl)
- 为不同组件提供合适的深度感

#### 圆角系统
- 从 xs (4px) 到 2xl (24px) 的圆角等级
- 更现代的圆角设计

#### 字体系统
- 完整的字体大小等级 (xs 到 4xl)
- 多种字重选项 (light 到 extrabold)
- 行高和字间距的精细控制

### 2. 新增 UI 组件

#### BaseModal (模态对话框)
- 替代原生 confirm 对话框
- 支持自定义标题、内容和按钮
- 优雅的进入/退出动画
- 支持背景点击关闭
- 响应式设计

**使用场景**:
- 删除确认
- 表单提交确认
- 重要操作提示

#### BaseCard (卡片)
- 统一的卡片容器组件
- 支持 header、body、footer 插槽
- 可选的悬停效果
- 玻璃态效果支持

**使用场景**:
- 内容分组
- 信息展示
- 功能模块容器

#### BaseBadge (徽章)
- 多种颜色变体 (primary, success, danger, warning, info)
- 三种尺寸 (sm, md, lg)
- 用于状态标识和标签

**使用场景**:
- 状态标识
- 数量显示
- 分类标签

#### StatCard (统计卡片)
- 专门用于展示统计数据
- 支持图标和渐变背景
- 悬停动画效果

**使用场景**:
- 仪表板统计
- 关键指标展示

#### EmptyState (空状态)
- 友好的空状态提示
- 支持自定义图标、标题和描述
- 可添加操作按钮

**使用场景**:
- 列表为空时
- 搜索无结果时
- 初始状态引导

### 3. 页面布局改进

#### 统计面板
在页面顶部添加了统计面板,展示:
- 代理规则总数
- 活跃规则数量
- 总请求数
- 系统状态

**优势**:
- 一目了然的数据概览
- 视觉吸引力强
- 提升专业感

#### 头部设计
- 更大的标题字体 (3xl)
- 渐变文字效果
- 优化的副标题
- 淡入动画

#### 卡片式布局
- 所有主要区域使用卡片容器
- 更大的圆角 (xl)
- 更强的阴影效果
- 悬停时的微交互

### 4. 组件优化

#### RuleList (规则列表)
- 使用 EmptyState 组件替代简单文本
- 更友好的空状态提示
- 引导用户添加第一条规则

#### RuleItem (规则项)
- 使用 BaseModal 替代原生 confirm
- 更详细的删除确认信息
- 优化的输入框样式
- 使用等宽字体显示 URL
- 改进的按钮样式和动画

#### RuleForm (规则表单)
- 使用 BaseInput 和 BaseButton 组件
- 统一的样式和交互
- 更好的加载状态显示

#### StatusMessage (状态消息)
- 添加图标指示器
- 玻璃态背景效果
- 优雅的进入/退出动画
- 更好的视觉层次

### 5. 动画和过渡

#### 新增动画
- `fadeIn` - 淡入动画
- `slideIn` - 滑入动画
- `slideUp` - 上滑动画
- `scaleIn` - 缩放动画
- `pulse` - 脉冲动画

#### 过渡效果
- 使用 cubic-bezier 缓动函数
- 三种速度等级 (fast, base, slow)
- 统一的过渡时长

#### 微交互
- 按钮悬停时上移 2px
- 卡片悬停时上移 2px
- 表格行悬停时轻微缩放
- 输入框聚焦时的阴影效果

### 6. 响应式设计

#### 断点系统
- 移动端: < 768px
- 小屏移动端: < 480px
- 平板: 769px - 1024px
- 桌面: > 1025px

#### 移动端优化
- 统计卡片单列布局
- 减小间距和字体大小
- 全屏模态对话框
- 优化的触摸目标大小

### 7. 可访问性改进

#### 颜色对比
- 所有文本都符合 WCAG AA 标准
- 清晰的视觉层次

#### 键盘导航
- 所有交互元素可通过键盘访问
- 清晰的焦点指示器

#### 语义化 HTML
- 正确使用 HTML5 语义标签
- 合理的标题层级

#### ARIA 支持
- 模态对话框的 aria-label
- 按钮的 title 属性

### 8. 性能优化

#### CSS 优化
- 使用 CSS 变量减少重复代码
- 模块化的样式文件
- 优化的选择器

#### 动画性能
- 使用 transform 和 opacity 进行动画
- 避免触发重排的属性
- 合理的动画时长

## 技术栈

- **Vue 3** - Composition API
- **Pinia** - 状态管理
- **CSS Variables** - 设计系统
- **CSS Modules** - 样式隔离

## 设计原则

### 1. 移动优先
从移动端开始设计,逐步增强到桌面端

### 2. 渐进增强
基础功能在所有设备上可用,高级特性逐步增强

### 3. 一致性
统一的设计语言和交互模式

### 4. 反馈
所有操作都有明确的视觉反馈

### 5. 容错性
友好的错误提示和空状态
