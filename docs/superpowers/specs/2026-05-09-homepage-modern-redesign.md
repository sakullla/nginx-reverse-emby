# 首页现代化重设计规范

**日期**: 2026-05-09
**目标**: 让首页更现代、有设计感、更简洁清爽
**设计方向**: 现代质感仪表板（基于 2025-2026 Dashboard UI 最佳实践）

---

## 1. 设计原则

基于行业最佳实践（[improvado.io](https://improvado.io/blog/dashboard-design-guide), [uxpin.com](https://www.uxpin.com/studio/blog/dashboard-design-principles/), [think.design](https://think.design/blog/dashboard-design-in-2026-dos-and-donts/)）：

- **40-30-20-10 空间规则**: 40% 屏幕给最重要指标，30% 给次要 KPI，20% 给趋势上下文，10% 给导航/表格
- **5-9 指标限制**: 主视图不超过 9 个核心指标，超过 12 个会导致 40% 参与度下降
- **30-40% 留白**: 防止认知过载，让信息自然呼吸
- **玻璃拟态克制使用**: 只在最高优先级元素上使用，次要元素用纯色背景（[uxpilot.ai](https://uxpilot.ai/blogs/glassmorphism-ui)）
- **F 型布局**: 最重要的 KPI 放在左上象限，最大的字体（2-3× 于次要指标）

## 2. 布局架构

### 2.1 当前布局问题
- 信息从上到下平铺，缺乏视觉层级
- 统计卡片与流量模块权重相同，没有主次之分
- 节点表格占据过多首屏空间

### 2.2 新布局：双层架构

```
┌─────────────────────────────────────────────┐
│  页面标题 + 副标题                            │
├──────────────────┬──────────────────────────┤
│                  │                          │
│  节点健康度卡片   │  规则概览卡片            │  ← 上层 (60%)
│  (L1 毛玻璃)      │  (L2 纯色)               │
│                  │                          │
├──────────────────┴──────────────────────────┤
│                                             │
│          流量趋势图 (全宽)                   │  ← 核心数据区
│                                             │
├──────────────────┬──────────────────────────┤
│  流量分布环形图   │  Top 规则/节点排行       │  ← 下层 (40%)
│                  │                          │
├──────────────────┴──────────────────────────┤
│                                             │
│          节点状态表格 (精简)                 │
│                                             │
└─────────────────────────────────────────────┘
```

### 2.3 统计卡合并策略

将 4 张独立统计卡合并为 2 张更具信息密度的卡片：

**节点健康度卡**（原在线+离线合并）：
- 主数值：`onlineCount / totalCount`，大字号展示
- 辅助信息：在线百分比进度条
- 状态指示：如果有离线节点，显示红色脉冲点；全部在线显示绿色稳定点
- 点击跳转：/agents

**规则概览卡**（原 HTTP+L4 合并）：
- 主数值：`rulesCount + l4Count`（总规则数）
- 辅助信息：HTTP 数量 / L4 数量分项显示
- 点击跳转：/rules

## 3. 视觉语言

### 3.1 卡片三级系统

| 级别 | 名称 | 视觉效果 | 用途 | 圆角 |
|------|------|----------|------|------|
| L1 | 焦点卡 | 毛玻璃：`backdrop-filter: blur(12px)` + `bg-white/5` + `border-white/10` | 节点健康度 | `radius-2xl` (16px) |
| L2 | 数据卡 | 纯色表面：`--color-bg-surface` + `border: 1.5px solid --color-border-default` | 流量统计、排行 | `radius-xl` (12px) |
| L3 | 容器 | 无边框，仅背景色 `bg-surface/50` 区分 | 节点表格 | `radius-lg` (8px) |

### 3.2 颜色层级

借鉴最佳实践的透明度层级系统：

| 内容类型 | 不透明度 | 用途 |
|----------|----------|------|
| 核心数字 | 100% | 统计卡主数值、关键指标 |
| 主要文字 | 85% | 卡片标题、重要标签 |
| 次要数据 | 60-70% | 辅助数值、描述文字 |
| 辅助标签 | 45-55% | 时间戳、单位、提示文字 |
| 背景边框 | 8-12% | 卡片边框、分隔线 |

### 3.3 排版层级

| 元素 | 字号 | 字重 | 其他 |
|------|------|------|------|
| 页面标题 | 24px | 700 | letter-spacing: -0.02em |
| 核心数字 | 36px | 600 | font-variant-numeric: tabular-nums |
| 卡片标题 | 14px | 600 | - |
| 正文/辅助 | 13px | 400 | - |
| 小标签 | 12px | 400 | text-transform: uppercase（可选） |

### 3.4 间距系统

- 页面内边距：24px（桌面）/ 16px（移动端）
- 区块间距：32px（原来 16-24px，增加留白）
- 卡片内边距：20px
- 卡片间隙：16px
- 元素间隙：12px

## 4. 微交互

### 4.1 统计卡 Hover
- 数字轻微放大：`transform: scale(1.02)`
- 底部强调色条滑入：`height: 0 → 3px`，`translateY` 从下方进入
- 过渡时间：`duration-normal` (200ms)

### 4.2 表格行 Hover
- 背景色过渡：`background-color` 变化，不使用边框变化
- 过渡时间：`duration-fast` (150ms)

### 4.3 入场动画
- 卡片错开入场：`fadeIn + translateY(8px → 0)`
- 错开间隔：50ms
- 总时长：300ms

### 4.4 状态指示
- 在线状态：绿色稳定圆点（不闪烁）
- 离线状态：红色脉冲动画（`animation: pulse 2s infinite`）
- 有告警时：卡片边框颜色过渡为警告色

## 5. 响应式

### 5.1 大屏（>1280px）
- 上层：2 列（节点健康度 + 规则概览）
- 中层：流量趋势图全宽
- 下层：2 列（分布图 + Top 排行）

### 5.2 中屏（768-1279px）
- 上层：2 列保持
- 下层：堆叠为单列

### 5.3 小屏（<768px）
- 全部堆叠为单列
- 图表简化（移除部分参考线）
- 表格列隐藏（隐藏 L4 数量、模式列）

## 6. 性能考虑

- `backdrop-filter` 在低端设备上可能影响性能，提供降级方案（无 blur 的半透明背景）
- 使用 `will-change: transform` 在动画元素上
- 图表使用 `requestAnimationFrame` 优化
- 入场动画仅在首屏可见元素上触发

## 8. 非目标（明确排除）

- 不改动流量图表的数据逻辑或 API 调用
- 不新增功能模块（保持现有功能范围）
- 不改路由结构或页面导航
- 不引入新的图表库（保持 ApexCharts）
