# 流量统计看板布局优化设计

## 背景

当前流量统计看板（`DashboardTrafficModule.vue`）采用三栏 + 底部四格统计卡片的布局。主要问题：
1. 信息密度不够高，关键数据分布太散
2. 右栏（Top 规则）在规则数量少时大量留白
3. 底部四格统计卡片纵向占用过多

## 目标

- 将统计信息合并到顶栏，减少纵向占用
- 右栏保持内容充实，不再空旷
- 趋势图纵向撑满，增强视觉重心
- 保持现有三栏布局结构

## 布局设计

### 顶栏（合并统计信息）

统计信息从底部移入顶栏右侧，与粒度切换 / 节点选择器并列：

```
| 流量统计 | 阻断: 1/3 | 已用/额度: 4.5G/6G [进度条] | 剩余: 1.5G | 小时 日 月 | 全部节点 ▾ |
```

统计项横向排列，字体略小（~12px），仅保留核心值（阻断数、已用/额度、剩余），计费周期移至左栏。

### 三栏内容区

| 左栏 (260px) | 中栏 (flex: 1) | 右栏 (280px) |
|---|---|---|
| 流量分布 (环形图) | 流量趋势 (纵向撑满) | 计费周期 |
| 计费周期 | | Top 节点 (5 条) |
| Top 节点 | | Top 规则 (5 条) |

### 各卡片说明

**左栏：**
1. **流量分布** — TrafficQuotaRing 环形图 + 图例，内容不变
2. **计费周期** — 从原底部统计卡片迁移，显示周期起始日 + 方向 + 日均流量
3. **Top 节点** — 排名列表，可点击跳转详情页

**中栏：**
- **流量趋势** — TrafficTrendChart，设置 `flex: 1` 纵向撑满剩余高度，不再受固定高度限制

**右栏：**
1. **计费周期** — 从原底部统计卡片迁移，显示周期起始日 + 方向 + 日均流量
2. **Top 节点** — 从原左栏迁移，5 条（原为 3-5 条），可点击跳转
3. **Top 规则** — 5 条带进度条（原为后端返回数量），可点击跳转

### 移除的内容

- 底部四格统计卡片行（阻断 / 计费周期 / 已用 / 剩余）全部移除
- 底部统计卡片的数据已合并到顶栏统计条（阻断、已用/额度、剩余）和左栏计费周期卡片

## 响应式

- `>1023px`: 三栏布局，顶栏统计信息横向排列
- `768-1023px`: 顶栏统计信息折行，三栏变两栏（趋势图全宽跨栏）
- `<768px`: 单栏堆叠，顶栏统计信息隐藏（趋势图上叠加显示）

## 文件变更

| 文件 | 变更 |
|---|---|
| `DashboardTrafficModule.vue` | 主要变更：顶栏添加统计条、移除底部统计行、重排三栏卡片、趋势图 flex 撑满、Top 规则上限改为 5 |

## 数据约束

- Top 规则：从后端 `top_rules` 数组取前 5 条
- Top 节点：从 `top_nodes` 或 `overviewAgents` 取前 5 条
- 趋势图高度：通过 `flex: 1` 自适应父容器剩余高度，不设固定 `min-height`
